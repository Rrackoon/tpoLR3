# Краткий отчет

## Тема

Функциональное тестирование интерфейса сайта `WolframAlpha` с помощью `Selenium`.

## Что сделано

1. Сформирован расширенный набор из 15 `Use Case`.
2. На их основе подготовлено покрытие из 23 тест-кейсов.
3. Реализованы автоматические UI-тесты на Java.
4. Основная Selenium-логика и page objects вынесены в `src/main/java/ru/itmo`.
5. Тесты оставлены в `src/test/java/ru/itmo` и логически разделены на несколько классов по категориям сценариев.

## Особенности сайта

- WolframAlpha активно использует динамическую генерацию интерфейса.
- Значительная часть результатов отображается как изображения с осмысленным `alt`, а не как обычный текст.
- Поэтому в проверках используются:
  - заголовки pod-блоков;
  - `alt` изображений внутри pod;
  - наличие графических элементов `img/svg/canvas`;
  - URL и заголовок страницы.

## Реализованные группы тестов

- основной функционал вычислений;
- разные типы запросов;
- негативные сценарии ввода;
- тесты интерфейса и поведения поля ввода;
- UX-сценарии повторного поиска и обновления страницы;
- кроссбраузерная проверка.

## Инструменты

- Java 21
- Maven
- Selenium 4
- JUnit 5

## Замечание по Selenium RC

В исходной формулировке задания указан `Selenium RC`, однако для современных версий `Chrome` и `Firefox` он считается устаревшим. Поэтому в проекте использован современный стек `Selenium WebDriver`, который фактически решает ту же задачу автоматизации и совместим с актуальными браузерами.

## Запуск
mvn test -Dbrowser=chrome
