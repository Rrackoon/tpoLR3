# Use Cases для WolframAlpha

## UC1. Простое вычисление

- Запрос: `2+2`
- Цель: получить базовый численный результат.
- Ожидаемый результат: отображается pod `Result:` со значением `4`.

## UC2. Сложное выражение

- Запрос: `integrate x^2`
- Цель: получить математический результат для интеграла.
- Ожидаемый результат: отображается pod `Indefinite integral:` с формулой результата.

## UC3. Решение уравнения

- Запрос: `solve x^2=4`
- Цель: получить корни уравнения.
- Ожидаемый результат: отображается pod `Result:` со значением `x = ± 2`.

## UC4. Построение графика

- Запрос: `plot x^2`
- Цель: получить графическое представление функции.
- Ожидаемый результат: отображается pod `Plot:` с визуальным графиком.

## UC5. Географический запрос

- Запрос: `capital of Germany`
- Цель: получить фактографическую информацию.
- Ожидаемый результат: отображается ответ `Berlin, Germany`.

## UC6. Погодный запрос

- Запрос: `weather Paris`
- Цель: получить текущую погоду и прогноз.
- Ожидаемый результат: отображаются погодные pod-блоки.

## UC7. Конвертация валют

- Запрос: `10 USD to EUR`
- Цель: получить курс и результат конвертации.
- Ожидаемый результат: отображается pod `Result:` с суммой в евро.

## UC8. Научный запрос

- Запрос: `speed of light`
- Цель: получить физическую константу.
- Ожидаемый результат: отображается pod `Value:` со значением в `m/s`.

## UC9. Пустой ввод

- Запрос: пустая строка
- Цель: проверить устойчивость интерфейса.
- Ожидаемый результат: сайт не падает, поле ввода остается доступным.

## UC10. Спецсимволы

- Запрос: `@#$`
- Цель: проверить обработку нестандартного ввода.
- Ожидаемый результат: сайт корректно открывает страницу запроса без сбоя.

## UC11. Очень длинный запрос

- Запрос: строка длиннее обычного пользовательского запроса
- Цель: проверить устойчивость к длинному вводу.
- Ожидаемый результат: сайт остается работоспособным, поле ввода не ломается.

## UC12. Быстрый повторный ввод

- Сценарий: пользователь быстро вводит новый запрос поверх старого.
- Цель: проверить устойчивость поисковой строки при повторных запросах.
- Ожидаемый результат: новый запрос принимается и результаты обновляются.

## UC13. Использование autocomplete

- Сценарий: пользователь вводит начало запроса.
- Цель: проверить появление автоподсказок.
- Ожидаемый результат: отображаются релевантные подсказки.

## UC14. Переход по результатам

- Сценарий: пользователь использует ссылку из блока результатов, если она доступна.
- Цель: проверить наличие и пригодность ссылки в pod-блоке.
- Ожидаемый результат: ссылка существует и содержит валидный URL.

## UC15. Обновление страницы после поиска

- Сценарий: пользователь обновляет страницу результатов.
- Цель: проверить сохранение состояния результата.
- Ожидаемый результат: запрос и найденный результат сохраняются после refresh.
